    Forum des exercices 61 à 63.4

  • Exercice 61 : Sauvegarde de l’état du jeu (en Java) ; la commande acceptera un nom de fichier en second mot.
    Deux solutions possibles :
    1. sauvegarder les éléments du jeu
    2. sauvegarder les commandes exécutées
      (avantage: simplicité - inconvénient: voir exercice suivant)
    Pour la première solution :
    • Recenser toutes les informations à sauvegarder pour pouvoir retrouver le jeu dans le même état après une interruption.
    • Vérifier que toutes les classes à sauvegarder possèdent bien une méthode toString().
    • Écrire une nouvelle méthode sauvegarder() appelant notamment toString() pour chaque élément à sauvegarder.
    • Tester en vérifiant que toutes les informations nécessaires sont bien sauvegardées.
    • Remarque : Cette solution peut aussi être implémentée en utilisant XML ou la sérialisation, mais ces deux technologies sortent du cadre d'une première année en programmation.
    Pour la seconde solution :
    • Sauvegarder au fur et à mesure toute commande exécutée
    La rubrique "Plus de technique"

  • Exercice 62 : Rechargement de l’état du jeu (en Java) ; la commande acceptera un nom de fichier en second mot.
    En fonction de la solution retenue à l'exercice précédent,
    Pour la première solution :
    • Écrire une nouvelle méthode recharger() qui devra recréer les objets nécessaires d’après les String lues dans le fichier.
    • Tester le jeu "exhaustivement" pour vérifier que tout fonctionne normalement.
    Pour la seconde solution :
    • Exécuter les commandes lues dans le fichier de sauvegarde comme pour un test.
      Attention ! Si certaines commandes sont prévues pour avoir un comportement (vraiment) aléatoire, il pourra être impossible de retrouver exactement l'état antérieur à la sauvegarde.
  • Exercice 62.1 : Tester l'arrêt et la reprise du jeu dans plusieurs situations, en continuant à jouer ou pas, en sortant du programme ou pas, etc.

  • Exercice 63 : Compléter 3 scénarios de test + tout retester (1 scénario directement gagnant + 1 qui explore toutes les pièces + 1 qui teste toutes les commandes)
  • Exercice 63.1 : Tout recompiler dans chaque paquetage : il ne doit y avoir aucun warning
  • Exercice 63.2 : Compléter le scénario pour que le jeu soit jouable (possibilités de gagner et de perdre)
  • Exercice 63.3 : Incorporer dans son jeu tous les éléments du scénario qui n'y sont pas déjà, y compris les dialogues et/ou énigmes/épreuves/combats.
  • Exercice 63.4 : Re-générer les 2 javadoc du projet comme à l'exercice 60.4, et les mettre à jour sur la page web du jeu.
